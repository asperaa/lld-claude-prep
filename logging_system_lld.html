Concept

Logging System — Overview & Requirements

What you need to know before writing a single line of code

A logging system is a cross-cutting concern that every production application needs. The goal is to record timestamped events (info, warnings, errors) and route them to the right destination — console, files, remote servers — with the right format and level of detail.

Interview tip: Always start by clarifying requirements before jumping into code. This shows maturity and structured thinking.

Functional Requirements

  • Support multiple log levels: DEBUG, INFO, WARNING, ERROR, CRITICAL
  • Multiple output destinations (console, file, remote)
  • Configurable message formatting
  • Filtering by level or content
  • Log rotation for file handlers
  • Timestamped entries with logger name

Non-Functional Requirements

  • Thread-safe (multiple threads log simultaneously)
  • High performance — logging must not slow app down
  • Extensible — add new handlers/formatters without changes
  • Singleton logger (single source of truth)
  • Fail-safe — logging errors must not crash the app
  • Minimal memory footprint

Class

LogLevel — Enum

Why an Enum? Because levels need strict ordering and identity comparison

Why Enum, not integer constants? Enums give you type safety, IDE autocompletion, and prevent invalid values like logger.log(99, "msg"). The integer .value gives ordering for comparisons.
loglevel.pypython
from enum import Enum, IntEnum

# Option 1: regular Enum — values for ordering
class LogLevel(Enum):
    DEBUG    = 1
    INFO     = 2
    WARNING  = 3
    ERROR    = 4
    CRITICAL = 5

# Option 2: IntEnum — allows direct < > comparisons without .value
class LogLevel(IntEnum):
    DEBUG    = 1
    INFO     = 2
    WARNING  = 3
    ERROR    = 4
    CRITICAL = 5

# With IntEnum you can do:
# LogLevel.ERROR > LogLevel.INFO  →  True (direct comparison)
# With plain Enum you need:       →  LogLevel.ERROR.value > LogLevel.INFO.value

# Usage in interview: always explain the trade-off
# IntEnum is more ergonomic; plain Enum is safer (harder to misuse)
Class

LogRecord — The Data Object

A single log event, packaged as an immutable-ish object

A LogRecord captures everything about one log event: the level, the message, when it happened, and where it came from. Handlers and Formatters receive this object and decide what to do with it.

logrecord.pypython
from dataclasses import dataclass, field
from datetime import datetime
import traceback

@dataclass
class LogRecord:
    level:       LogLevel
    message:     str
    timestamp:   datetime = field(default_factory=datetime.now)
    logger_name: str      = "root"
    module:      str      = ""           # e.g. "user_service"
    line_no:     int      = 0
    exc_info:    str      = ""           # stack trace if exception provided

    def __post_init__(self):
        # auto-capture stack trace if message is an Exception
        if isinstance(self.message, Exception):
            self.exc_info = traceback.format_exc()
            self.message  = str(self.message)

    def __str__(self):
        base = (
            f"[{self.timestamp.strftime('%Y-%m-%d %H:%M:%S')}] "
            f"[{self.level.name:<8}] "        # left-aligned, 8 wide
            f"[{self.logger_name}] {self.message}"
        )
        if self.exc_info:
            base += f"\n{self.exc_info}"
        return base
Interview tip: Interviewers often ask "what fields does LogRecord have?" — level, message, timestamp, and logger_name are the minimum. Bonus points for exc_info (stack trace capture), module, and line_no.
Pattern: Singleton

Logger — Singleton Pattern

One logger to rule them all. Thread-safe via double-checked locking.

Why Singleton? If every class created its own logger, you'd have disconnected log streams with no central control over levels, handlers, or format. One Singleton means one consistent stream across the whole application.
logger.pypython
import threading
from typing import List

class Logger:
    """Thread-safe Singleton Logger."""

    _instance: "Logger"  = None
    _lock:     threading.Lock = threading.Lock()

    # ── Singleton constructor ──────────────────────────────────
    def __new__(cls) -> "Logger":
        if cls._instance is None:          # first check (no lock, fast path)
            with cls._lock:               # acquire lock
                if cls._instance is None: # second check (inside lock)
                    cls._instance = super().__new__(cls)
                    cls._instance._init()
        return cls._instance

    def _init(self):
        """Called once, the first time the singleton is created."""
        self._handlers:  List[Handler] = []
        self._min_level: LogLevel     = LogLevel.DEBUG
        self._name:      str            = "root"

    # ── Configuration ─────────────────────────────────────────
    def set_level(self, level: LogLevel):
        """Set the minimum level. Anything below this is silently dropped.
        e.g. set_level(WARNING) → DEBUG and INFO are ignored globally.
        """
        self._min_level = level

    def add_handler(self, handler: "Handler"):
        if handler not in self._handlers:  # prevent duplicate handlers
            self._handlers.append(handler)

    def remove_handler(self, handler: "Handler"):
        if handler in self._handlers:
            self._handlers.remove(handler)

    def clear_handlers(self):
        """Useful for testing — reset to clean state."""
        self._handlers.clear()

    # ── Core log method ───────────────────────────────────────
    def log(self, level: LogLevel, message: str):
        # Step 1: global level gate
        if level.value < self._min_level.value:
            return
        # Step 2: create the record
        record = LogRecord(
            level       = level,
            message     = message,
            logger_name = self._name,
        )
        # Step 3: notify all handlers (Observer pattern)
        for handler in self._handlers:
            try:
                handler.handle(record)
            except Exception as e:
                # CRITICAL: logging must NEVER crash your app
                print(f"[Logger internal error] {e}", file=__import__('sys').stderr)

    # ── Convenience methods ────────────────────────────────────
    def debug(self, msg: str):    self.log(LogLevel.DEBUG,    msg)
    def info(self, msg: str):     self.log(LogLevel.INFO,     msg)
    def warning(self, msg: str):  self.log(LogLevel.WARNING,  msg)
    def error(self, msg: str):    self.log(LogLevel.ERROR,    msg)
    def critical(self, msg: str): self.log(LogLevel.CRITICAL, msg)

    def exception(self, msg: str):
        """Log ERROR with current exception stack trace attached."""
        import traceback
        self.log(LogLevel.ERROR, msg + "\n" + traceback.format_exc())
Double-Checked Locking Explained: Without the inner check, two threads could both pass the outer if _instance is None, then queue up at the lock. The first creates the instance. The second, now inside the lock, would create a SECOND instance without the inner check. The double-check prevents this. This is a classic concurrent programming interview topic.
Pattern: Strategy

Formatters — Strategy Pattern

Swap how logs look without changing how they're stored or delivered

Why Strategy? Different destinations need different formats. Your console might want readable colored text. Your remote log aggregator wants JSON. Your file might want CSV. By making Formatter swappable, each Handler gets its own formatting strategy without any if/else chains.
formatters.pypython
from abc import ABC, abstractmethod
import json

# ────────────────────────────────────────────────────────
# Abstract Strategy
# ────────────────────────────────────────────────────────
class Formatter(ABC):
    @abstractmethod
    def format(self, record: LogRecord) -> str:
        """Convert a LogRecord into a string for output."""
        ...

# ────────────────────────────────────────────────────────
# Concrete Strategy 1: Plain Text
# ────────────────────────────────────────────────────────
class PlainTextFormatter(Formatter):
    def __init__(self, datefmt: str = "%Y-%m-%d %H:%M:%S"):
        self.datefmt = datefmt

    def format(self, record: LogRecord) -> str:
        ts = record.timestamp.strftime(self.datefmt)
        return f"[{ts}] [{record.level.name:<8}] [{record.logger_name}] {record.message}"

# Output example:
# [2024-01-15 14:32:10] [ERROR   ] [root] DB connection failed

# ────────────────────────────────────────────────────────
# Concrete Strategy 2: JSON (for log aggregators like ELK, Splunk)
# ────────────────────────────────────────────────────────
class JSONFormatter(Formatter):
    def format(self, record: LogRecord) -> str:
        payload = {
            "timestamp":   record.timestamp.isoformat(),
            "level":       record.level.name,
            "logger":      record.logger_name,
            "message":     record.message,
        }
        if record.exc_info:
            payload["exception"] = record.exc_info
        return json.dumps(payload)

# Output example:
# {"timestamp":"2024-01-15T14:32:10","level":"ERROR","logger":"root","message":"..."}

# ────────────────────────────────────────────────────────
# Concrete Strategy 3: Colored Console (ANSI escape codes)
# ────────────────────────────────────────────────────────
ANSI = {
    LogLevel.DEBUG:    "\033[36m",   # Cyan
    LogLevel.INFO:     "\033[32m",   # Green
    LogLevel.WARNING:  "\033[33m",   # Yellow
    LogLevel.ERROR:    "\033[31m",   # Red
    LogLevel.CRITICAL: "\033[35;1m", # Bold Magenta
}
RESET = "\033[0m"

class ColoredFormatter(Formatter):
    def format(self, record: LogRecord) -> str:
        color = ANSI.get(record.level, "")
        ts    = record.timestamp.strftime("%H:%M:%S")  # shorter for console
        return f"{color}[{ts}] [{record.level.name:<8}]{RESET} {record.message}"

# ────────────────────────────────────────────────────────
# Concrete Strategy 4: CSV (for data analysis)
# ────────────────────────────────────────────────────────
class CSVFormatter(Formatter):
    def format(self, record: LogRecord) -> str:
        msg = record.message.replace(",", ";")  # escape commas
        return f"{record.timestamp.isoformat()},{record.level.name},{record.logger_name},{msg}"

# ────────────────────────────────────────────────────────
# ★ Adding a new formatter = just create a new subclass.
#   Zero changes to Logger, Handler, or any existing code.
#   This is the Open/Closed Principle (OCP) in action.
# ────────────────────────────────────────────────────────
Pattern: Chain of Responsibility

Filters — Chain of Responsibility

Each filter checks one thing. Pass the record down the chain — or drop it.

Why Chain of Responsibility? A record must pass multiple independent checks before being logged. Chain of Responsibility lets you compose these checks without coupling them. You can add, remove, or reorder filters without touching existing filter code.
LogRecord
→
LevelFilter ≥ WARNING?
→
KeywordFilter "healthcheck"?
→
RateFilter ≤ 100/min?
→
✓ Accepted
filters.pypython
from abc import ABC, abstractmethod
from collections import deque
from datetime import datetime, timedelta

# ────────────────────────────────────────────────────────
# Abstract base: defines the chain protocol
# ────────────────────────────────────────────────────────
class Filter(ABC):
    def __init__(self):
        self._next: "Filter" = None

    def set_next(self, next_filter: "Filter") -> "Filter":
        """Set next filter and return it (enables fluent chaining).
        f1.set_next(f2).set_next(f3) → chain: f1 → f2 → f3
        """
        self._next = next_filter
        return next_filter  # return next so we can chain .set_next() calls

    def passes(self, record: LogRecord) -> bool:
        """Ask this filter, and if it passes, ask the next one."""
        if self._check(record):
            return self._next.passes(record) if self._next else True
        return False  # short-circuit: reject immediately

    @abstractmethod
    def _check(self, record: LogRecord) -> bool:
        """Implement your specific filter logic here."""
        ...

# ────────────────────────────────────────────────────────
# Concrete Filter 1: Level
# ────────────────────────────────────────────────────────
class LevelFilter(Filter):
    def __init__(self, min_level: LogLevel):
        super().__init__()
        self.min_level = min_level

    def _check(self, record: LogRecord) -> bool:
        return record.level.value >= self.min_level.value

# ────────────────────────────────────────────────────────
# Concrete Filter 2: Keyword blocklist (e.g. suppress noisy health checks)
# ────────────────────────────────────────────────────────
class KeywordFilter(Filter):
    def __init__(self, blocked_keywords: list):
        super().__init__()
        self.blocked = [kw.lower() for kw in blocked_keywords]

    def _check(self, record: LogRecord) -> bool:
        msg = record.message.lower()
        return not any(kw in msg for kw in self.blocked)

# ────────────────────────────────────────────────────────
# Concrete Filter 3: Rate limiter (max N logs per minute)
# ────────────────────────────────────────────────────────
class RateLimitFilter(Filter):
    """Uses a sliding window (deque of timestamps) to enforce rate limit."""
    def __init__(self, max_per_minute: int = 100):
        super().__init__()
        self.max_per_minute = max_per_minute
        self._timestamps    = deque()  # stores datetime of each recent log

    def _check(self, record: LogRecord) -> bool:
        now      = datetime.now()
        cutoff   = now - timedelta(minutes=1)
        # Remove old timestamps outside the 1-minute window
        while self._timestamps and self._timestamps[0] < cutoff:
            self._timestamps.popleft()
        if len(self._timestamps) >= self.max_per_minute:
            return False  # rate limit exceeded
        self._timestamps.append(now)
        return True

# ────────────────────────────────────────────────────────
# Concrete Filter 4: Module allow-list (only log from specific modules)
# ────────────────────────────────────────────────────────
class ModuleFilter(Filter):
    def __init__(self, allowed_modules: list):
        super().__init__()
        self.allowed = set(allowed_modules)

    def _check(self, record: LogRecord) -> bool:
        return record.logger_name in self.allowed

# ────────────────────────────────────────────────────────
# Building a chain (example)
# ────────────────────────────────────────────────────────
lf = LevelFilter(LogLevel.WARNING)
kf = KeywordFilter(["healthcheck", "ping"])
rf = RateLimitFilter(max_per_minute=50)

# Fluent chaining:
lf.set_next(kf).set_next(rf)  # record must pass lf → kf → rf
Pattern: Observer

Handlers — Observer Pattern

Logger is the subject. Handlers are observers. Each decides independently what to do.

Why Observer? Logger should not know HOW logs are written. It just notifies all registered handlers. Each handler independently decides how to handle the record. Adding a new destination = add a new handler. Zero changes to Logger.
handlers.pypython
from abc import ABC, abstractmethod
import os, sys, urllib.request, threading

# ────────────────────────────────────────────────────────
# Abstract Handler (Observer interface)
# ────────────────────────────────────────────────────────
class Handler(ABC):
    def __init__(self,
                 formatter:    "Formatter" = None,
                 filter_chain: "Filter"    = None):
        self.formatter    = formatter    or PlainTextFormatter()
        self.filter_chain = filter_chain  # None = accept everything

    def handle(self, record: LogRecord):
        """Called by Logger for every log event.
        1. Run filters  2. Format  3. Write"""
        if self.filter_chain and not self.filter_chain.passes(record):
            return                      # filtered out
        formatted = self.formatter.format(record)
        self._write(formatted)

    @abstractmethod
    def _write(self, message: str):
        """Subclasses implement this: where does the message go?"""
        ...

    def close(self):
        """Optional cleanup (e.g. flush + close file handles)."""
        pass

# ────────────────────────────────────────────────────────
# Concrete Handler 1: Console
# ────────────────────────────────────────────────────────
class ConsoleHandler(Handler):
    def __init__(self, stream=None, **kwargs):
        super().__init__(**kwargs)
        # Allow redirecting: stdout for info, stderr for errors
        self.stream = stream or sys.stdout

    def _write(self, message: str):
        print(message, file=self.stream, flush=True)

# ────────────────────────────────────────────────────────
# Concrete Handler 2: File
# ────────────────────────────────────────────────────────
class FileHandler(Handler):
    def __init__(self, filepath: str, encoding: str = "utf-8", **kwargs):
        super().__init__(**kwargs)
        self.filepath = filepath
        self.encoding = encoding
        # Create parent directory if it doesn't exist
        os.makedirs(os.path.dirname(os.path.abspath(filepath)), exist_ok=True)

    def _write(self, message: str):
        # Open in append mode each time (simple, safe, no state)
        with open(self.filepath, "a", encoding=self.encoding) as f:
            f.write(message + "\n")

# ────────────────────────────────────────────────────────
# Concrete Handler 3: Remote HTTP endpoint
# ────────────────────────────────────────────────────────
class RemoteHandler(Handler):
    def __init__(self, endpoint: str, timeout: int = 2, **kwargs):
        super().__init__(**kwargs)
        self.endpoint = endpoint
        self.timeout  = timeout

    def _write(self, message: str):
        try:
            data    = message.encode("utf-8")
            headers = {"Content-Type": "application/json"}
            req     = urllib.request.Request(
                self.endpoint, data=data, headers=headers, method="POST"
            )
            urllib.request.urlopen(req, timeout=self.timeout)
        except Exception as e:
            # NEVER let network failures crash the application
            print(f"[RemoteHandler failed] {e}", file=sys.stderr)

# ────────────────────────────────────────────────────────
# Concrete Handler 4: Database (e.g. audit trail)
# ────────────────────────────────────────────────────────
class DatabaseHandler(Handler):
    """Stores logs in a database table — useful for audit logs."""
    def __init__(self, db_connection, **kwargs):
        super().__init__(**kwargs)
        self.conn = db_connection

    def _write(self, message: str):
        try:
            cursor = self.conn.cursor()
            cursor.execute(
                "INSERT INTO logs (timestamp, message) VALUES (NOW(), ?)",
                (message,)
            )
            self.conn.commit()
        except Exception as e:
            print(f"[DatabaseHandler failed] {e}", file=sys.stderr)
Pattern: Decorator

RotatingFileHandler — Decorator Pattern

Wrap FileHandler behavior. Add rotation without modifying original class.

Why Decorator? We want to add file rotation to FileHandler without modifying it (Open/Closed Principle). Decorator wraps the original and adds new behavior — RotatingFileHandler IS a Handler but also knows how to rotate.
rotating_handler.pypython
import os, threading
from datetime import datetime, timedelta

# ────────────────────────────────────────────────────────
# Size-based rotating file handler
# ────────────────────────────────────────────────────────
class RotatingFileHandler(Handler):
    """Rotates log file when it exceeds max_bytes.
    Keeps backup_count old copies: app.log, app.log.1, app.log.2, ...
    """
    def __init__(self,
                 filepath:     str,
                 max_bytes:    int = 5_000_000,   # 5 MB default
                 backup_count: int = 5,
                 **kwargs):
        super().__init__(**kwargs)
        self.filepath     = filepath
        self.max_bytes    = max_bytes
        self.backup_count = backup_count
        self._lock        = threading.Lock()  # thread-safe rotation
        os.makedirs(os.path.dirname(os.path.abspath(filepath)), exist_ok=True)

    def _should_rotate(self) -> bool:
        return (
            os.path.exists(self.filepath) and
            os.path.getsize(self.filepath) >= self.max_bytes
        )

    def _do_rotate(self):
        """Shift existing backups: .4→.5, .3→.4, ..., log→.1"""
        for i in range(self.backup_count - 1, 0, -1):
            src = f"{self.filepath}.{i}"
            dst = f"{self.filepath}.{i+1}"
            if os.path.exists(src):
                os.rename(src, dst)
        if os.path.exists(self.filepath):
            os.rename(self.filepath, f"{self.filepath}.1")

    def _write(self, message: str):
        with self._lock:                   # lock so only one thread rotates
            if self._should_rotate():
                self._do_rotate()
            with open(self.filepath, "a", encoding="utf-8") as f:
                f.write(message + "\n")

# ────────────────────────────────────────────────────────
# Time-based rotating file handler (daily rotation)
# ────────────────────────────────────────────────────────
class TimedRotatingFileHandler(Handler):
    """Rotates at midnight each day. Creates files like app.log.2024-01-15"""
    def __init__(self, filepath: str, backup_days: int = 30, **kwargs):
        super().__init__(**kwargs)
        self.filepath    = filepath
        self.backup_days = backup_days
        self._today      = datetime.now().date()
        self._lock       = threading.Lock()
        os.makedirs(os.path.dirname(os.path.abspath(filepath)), exist_ok=True)

    def _current_filepath(self) -> str:
        # e.g.  app.log.2024-01-15
        return f"{self.filepath}.{self._today}"

    def _cleanup_old_logs(self):
        """Delete log files older than backup_days."""
        cutoff   = datetime.now().date() - timedelta(days=self.backup_days)
        directory = os.path.dirname(os.path.abspath(self.filepath))
        base      = os.path.basename(self.filepath)
        for fname in os.listdir(directory):
            if fname.startswith(base + "."):
                date_part = fname[len(base)+1:]
                try:
                    fdate = datetime.strptime(date_part, "%Y-%m-%d").date()
                    if fdate < cutoff:
                        os.remove(os.path.join(directory, fname))
                except ValueError:
                    pass

    def _write(self, message: str):
        with self._lock:
            today = datetime.now().date()
            if today != self._today:      # new day detected
                self._today = today
                self._cleanup_old_logs()   # remove stale files
            with open(self._current_filepath(), "a", encoding="utf-8") as f:
                f.write(message + "\n")
Pattern: Factory

LoggerFactory — Factory Pattern

Hide wiring complexity. Callers just say what kind of logger they want.

Why Factory? Setting up a production logger requires wiring 3+ handlers, each with its own formatter and filter chain. That's 10–15 lines of boilerplate. A factory encapsulates this and gives callers a single clean method call.
logger_factory.pypython
class LoggerFactory:

    @staticmethod
    def create_console_logger(min_level: LogLevel = LogLevel.DEBUG) -> Logger:
        """Colored console logger for local development."""
        logger = Logger()
        logger.clear_handlers()
        logger.set_level(min_level)
        logger.add_handler(ConsoleHandler(formatter=ColoredFormatter()))
        return logger

    @staticmethod
    def create_file_logger(filepath: str,
                            min_level: LogLevel = LogLevel.DEBUG) -> Logger:
        """Plain text file logger — good for audit trails."""
        logger = Logger()
        logger.clear_handlers()
        logger.add_handler(
            FileHandler(filepath, formatter=PlainTextFormatter(),
                        filter_chain=LevelFilter(min_level))
        )
        return logger

    @staticmethod
    def create_production_logger(log_dir: str = "logs") -> Logger:
        """
        Full production setup:
          • Console   → all levels   → Colored text
          • File      → WARNING+     → Plain text, size-rotating
          • Remote    → ERROR+       → JSON, fail-safe
        """
        logger = Logger()
        logger.clear_handlers()
        logger.set_level(LogLevel.DEBUG)

        # 1. Console: colored, all levels (no filter needed)
        logger.add_handler(ConsoleHandler(formatter=ColoredFormatter()))

        # 2. Rotating file: WARNING+, plain text
        rotating = RotatingFileHandler(
            filepath     = os.path.join(log_dir, "app.log"),
            max_bytes    = 10_000_000,   # 10 MB per file
            backup_count = 5,
            formatter    = PlainTextFormatter(),
            filter_chain = LevelFilter(LogLevel.WARNING),
        )
        logger.add_handler(rotating)

        # 3. Remote error sink: ERROR+, JSON, rate-limited
        rate_then_level = RateLimitFilter(max_per_minute=200)
        rate_then_level.set_next(LevelFilter(LogLevel.ERROR))
        remote = RemoteHandler(
            endpoint     = "https://logs.example.com/ingest",
            formatter    = JSONFormatter(),
            filter_chain = rate_then_level,
        )
        logger.add_handler(remote)

        return logger

    @staticmethod
    def create_test_logger() -> Logger:
        """Logger for unit tests: captures logs in memory, not disk/console.
        Useful to assert that certain log messages were produced."""
        logger = Logger()
        logger.clear_handlers()
        logger.add_handler(InMemoryHandler())  # see advanced section
        return logger
Advanced

Thread Safety

Logging from multiple threads is the default case in production — handle it correctly

Common interview question: "What happens if two threads call logger.log() simultaneously?" — you need to know exactly where race conditions can occur.
thread_safe_handlers.pypython
import threading

# ── Race condition 1: Singleton creation ────────────────
# Solved by double-checked locking in Logger.__new__() already shown above.

# ── Race condition 2: File handler (two threads write simultaneously) ────
class ThreadSafeFileHandler(Handler):
    def __init__(self, filepath: str, **kwargs):
        super().__init__(**kwargs)
        self.filepath  = filepath
        self._lock     = threading.Lock()  # one write at a time

    def _write(self, message: str):
        with self._lock:
            with open(self.filepath, "a") as f:
                f.write(message + "\n")

# ── Race condition 3: Handler list modification while iterating ─────────
# In Logger.log() we iterate self._handlers.
# If another thread calls add_handler() simultaneously → RuntimeError
# Fix: use a threading.RLock (reentrant) around handler list access

class Logger:   # extended version
    def _init(self):
        self._handlers  = []
        self._lock      = threading.RLock()  # RLock = reentrant
        self._min_level = LogLevel.DEBUG

    def add_handler(self, handler):
        with self._lock:
            if handler not in self._handlers:
                self._handlers.append(handler)

    def log(self, level, message):
        if level.value < self._min_level.value:
            return
        record = LogRecord(level, message)
        with self._lock:
            handlers_snapshot = list(self._handlers)  # copy to release lock fast
        for handler in handlers_snapshot:
            try:
                handler.handle(record)
            except Exception as e:
                print(f"[Logger error] {e}", file=sys.stderr)

# ── Why snapshot the handler list? ─────────────────────────────────────
# We copy self._handlers while holding the lock, then release it before
# calling handle(). This avoids holding the lock during I/O (which could
# be slow), preventing other threads from blocking unnecessarily.
Advanced

Async-Safe Logging — Queue Handler

Never block your main thread on disk I/O or network calls

Why async logging? Writing to disk or sending HTTP takes milliseconds. If the main thread blocks on every log call, a high-throughput service slows to a crawl. Solution: log calls just enqueue a record in memory (microseconds), and a background worker thread drains the queue.
async_handler.pypython
import queue, threading, sys

# ────────────────────────────────────────────────────────
# QueueHandler: enqueues records (non-blocking, O(1))
# ────────────────────────────────────────────────────────
class QueueHandler(Handler):
    """Wraps any handler. log() returns instantly; background thread writes."""

    def __init__(self, target_handler: Handler, maxsize: int = 10_000):
        super().__init__()
        self.target  = target_handler
        self._queue  = queue.Queue(maxsize=maxsize)
        self._stop   = threading.Event()
        self._worker = threading.Thread(
            target=self._drain, daemon=True, name="log-worker"
        )
        self._worker.start()

    def _write(self, message: str):
        try:
            self._queue.put_nowait(message)  # non-blocking put
        except queue.Full:
            # Queue is full — drop the log rather than block the app
            print("[QueueHandler] queue full, log dropped", file=sys.stderr)

    def _drain(self):
        """Background worker: pulls from queue, passes to real handler."""
        while not self._stop.is_set():
            try:
                message = self._queue.get(timeout=0.1)
                self.target._write(message)
                self._queue.task_done()
            except queue.Empty:
                continue

    def close(self):
        """Graceful shutdown: flush remaining logs, then stop."""
        self._queue.join()           # wait for queue to empty
        self._stop.set()
        self._worker.join()
        self.target.close()

# ── Usage ───────────────────────────────────────────────
# Wrap any slow handler with QueueHandler:
file_handler  = FileHandler("app.log", formatter=PlainTextFormatter())
async_handler = QueueHandler(file_handler)
logger        = Logger()
logger.add_handler(async_handler)

# logger.error("...") returns instantly — disk write happens in background

# ────────────────────────────────────────────────────────
# InMemoryHandler: for unit testing
# ────────────────────────────────────────────────────────
class InMemoryHandler(Handler):
    """Stores all log messages in a list — great for assertions in tests."""
    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self.records: list = []

    def _write(self, message: str):
        self.records.append(message)

    def clear(self):
        self.records.clear()

    def contains(self, substr: str) -> bool:
        return any(substr in r for r in self.records)

"""
Logging System — Complete LLD Implementation
Patterns: Singleton, Strategy, Chain of Responsibility, Observer,
          Decorator, Factory
"""

from __future__ import annotations
from abc         import ABC, abstractmethod
from collections import deque
from dataclasses import dataclass, field
from datetime    import datetime, timedelta
from enum        import IntEnum
import json, os, queue, sys, threading, traceback, urllib.request

# ═══════════════════════════════════════════════════════════
# 1. LogLevel
# ═══════════════════════════════════════════════════════════
class LogLevel(IntEnum):
    DEBUG    = 1
    INFO     = 2
    WARNING  = 3
    ERROR    = 4
    CRITICAL = 5

# ═══════════════════════════════════════════════════════════
# 2. LogRecord
# ═══════════════════════════════════════════════════════════
@dataclass
class LogRecord:
    level:       LogLevel
    message:     str
    timestamp:   datetime = field(default_factory=datetime.now)
    logger_name: str      = "root"
    exc_info:    str      = ""

    def __post_init__(self):
        if isinstance(self.message, Exception):
            self.exc_info = traceback.format_exc()
            self.message  = str(self.message)

# ═══════════════════════════════════════════════════════════
# 3. Formatters (Strategy Pattern)
# ═══════════════════════════════════════════════════════════
class Formatter(ABC):
    @abstractmethod
    def format(self, record: LogRecord) -> str: ...

class PlainTextFormatter(Formatter):
    def format(self, record: LogRecord) -> str:
        ts = record.timestamp.strftime("%Y-%m-%d %H:%M:%S")
        return f"[{ts}] [{record.level.name:<8}] [{record.logger_name}] {record.message}"

class JSONFormatter(Formatter):
    def format(self, record: LogRecord) -> str:
        d = {"timestamp": record.timestamp.isoformat(),
             "level": record.level.name, "message": record.message}
        if record.exc_info: d["exception"] = record.exc_info
        return json.dumps(d)

ANSI_COLORS = {
    LogLevel.DEBUG:    "\033[36m", LogLevel.INFO:     "\033[32m",
    LogLevel.WARNING:  "\033[33m", LogLevel.ERROR:    "\033[31m",
    LogLevel.CRITICAL: "\033[35;1m",
}
RESET = "\033[0m"

class ColoredFormatter(Formatter):
    def format(self, record: LogRecord) -> str:
        c  = ANSI_COLORS.get(record.level, "")
        ts = record.timestamp.strftime("%H:%M:%S")
        return f"{c}[{ts}][{record.level.name:<8}]{RESET} {record.message}"

# ═══════════════════════════════════════════════════════════
# 4. Filters (Chain of Responsibility)
# ═══════════════════════════════════════════════════════════
class Filter(ABC):
    def __init__(self): self._next: Filter = None
    def set_next(self, nxt: Filter) -> Filter:
        self._next = nxt; return nxt
    def passes(self, r: LogRecord) -> bool:
        return (self._next.passes(r) if self._next else True) if self._check(r) else False
    @abstractmethod
    def _check(self, r: LogRecord) -> bool: ...

class LevelFilter(Filter):
    def __init__(self, min_level: LogLevel):
        super().__init__(); self.min_level = min_level
    def _check(self, r: LogRecord) -> bool:
        return r.level >= self.min_level

class KeywordFilter(Filter):
    def __init__(self, blocked: list):
        super().__init__(); self.blocked = [k.lower() for k in blocked]
    def _check(self, r: LogRecord) -> bool:
        return not any(k in r.message.lower() for k in self.blocked)

class RateLimitFilter(Filter):
    def __init__(self, max_per_min: int = 100):
        super().__init__(); self.max = max_per_min; self._ts = deque()
    def _check(self, r: LogRecord) -> bool:
        now = datetime.now(); cutoff = now - timedelta(minutes=1)
        while self._ts and self._ts[0] < cutoff: self._ts.popleft()
        if len(self._ts) >= self.max: return False
        self._ts.append(now); return True

# ═══════════════════════════════════════════════════════════
# 5. Handlers (Observer Pattern)
# ═══════════════════════════════════════════════════════════
class Handler(ABC):
    def __init__(self, formatter: Formatter = None, filter_chain: Filter = None):
        self.formatter    = formatter or PlainTextFormatter()
        self.filter_chain = filter_chain
    def handle(self, record: LogRecord):
        if self.filter_chain and not self.filter_chain.passes(record): return
        self._write(self.formatter.format(record))
    @abstractmethod
    def _write(self, msg: str): ...
    def close(self): pass

class ConsoleHandler(Handler):
    def _write(self, msg: str): print(msg, flush=True)

class FileHandler(Handler):
    def __init__(self, filepath: str, **kw):
        super().__init__(**kw); self.filepath = filepath
        os.makedirs(os.path.dirname(os.path.abspath(filepath)), exist_ok=True)
    def _write(self, msg: str):
        with open(self.filepath, "a") as f: f.write(msg + "\n")

class RotatingFileHandler(Handler):
    def __init__(self, filepath, max_bytes=5_000_000, backup_count=5, **kw):
        super().__init__(**kw)
        self.filepath = filepath; self.max_bytes = max_bytes
        self.backup_count = backup_count; self._lock = threading.Lock()
        os.makedirs(os.path.dirname(os.path.abspath(filepath)), exist_ok=True)
    def _should_rotate(self):
        return os.path.exists(self.filepath) and os.path.getsize(self.filepath) >= self.max_bytes
    def _do_rotate(self):
        for i in range(self.backup_count-1, 0, -1):
            s, d = f"{self.filepath}.{i}", f"{self.filepath}.{i+1}"
            if os.path.exists(s): os.rename(s, d)
        if os.path.exists(self.filepath): os.rename(self.filepath, f"{self.filepath}.1")
    def _write(self, msg: str):
        with self._lock:
            if self._should_rotate(): self._do_rotate()
            with open(self.filepath, "a") as f: f.write(msg + "\n")

class InMemoryHandler(Handler):
    def __init__(self, **kw): super().__init__(**kw); self.records = []
    def _write(self, msg): self.records.append(msg)
    def contains(self, s): return any(s in r for r in self.records)

class QueueHandler(Handler):
    def __init__(self, target: Handler, maxsize=10_000):
        super().__init__(); self.target = target; self._q = queue.Queue(maxsize)
        self._stop = threading.Event()
        self._t = threading.Thread(target=self._drain, daemon=True); self._t.start()
    def _write(self, msg):
        try: self._q.put_nowait(msg)
        except queue.Full: print("[QueueHandler] dropped", file=sys.stderr)
    def _drain(self):
        while not self._stop.is_set():
            try: msg = self._q.get(timeout=0.1); self.target._write(msg); self._q.task_done()
            except queue.Empty: continue
    def close(self): self._q.join(); self._stop.set(); self._t.join(); self.target.close()

# ═══════════════════════════════════════════════════════════
# 6. Logger (Singleton + Observer subject)
# ═══════════════════════════════════════════════════════════
class Logger:
    _instance = None; _class_lock = threading.Lock()

    def __new__(cls):
        if cls._instance is None:
            with cls._class_lock:
                if cls._instance is None:
                    cls._instance = super().__new__(cls); cls._instance._init()
        return cls._instance

    def _init(self):
        self._handlers = []; self._lock = threading.RLock()
        self._min_level = LogLevel.DEBUG; self._name = "root"

    def set_level(self, l): self._min_level = l
    def add_handler(self, h):
        with self._lock:
            if h not in self._handlers: self._handlers.append(h)
    def remove_handler(self, h):
        with self._lock:
            if h in self._handlers: self._handlers.remove(h)
    def clear_handlers(self):
        with self._lock: self._handlers.clear()

    def log(self, level: LogLevel, message: str):
        if level < self._min_level: return
        record = LogRecord(level=level, message=message, logger_name=self._name)
        with self._lock: snapshot = list(self._handlers)
        for h in snapshot:
            try: h.handle(record)
            except Exception as e: print(f"[Logger err] {e}", file=sys.stderr)

    def debug(self, m):    self.log(LogLevel.DEBUG, m)
    def info(self, m):     self.log(LogLevel.INFO, m)
    def warning(self, m):  self.log(LogLevel.WARNING, m)
    def error(self, m):    self.log(LogLevel.ERROR, m)
    def critical(self, m): self.log(LogLevel.CRITICAL, m)
    def exception(self, m):
        self.log(LogLevel.ERROR, m + "\n" + traceback.format_exc())

# ═══════════════════════════════════════════════════════════
# 7. LoggerFactory (Factory Pattern)
# ═══════════════════════════════════════════════════════════
class LoggerFactory:
    @staticmethod
    def create_console_logger(level=LogLevel.DEBUG):
        l = Logger(); l.clear_handlers(); l.set_level(level)
        l.add_handler(ConsoleHandler(formatter=ColoredFormatter())); return l

    @staticmethod
    def create_production_logger(log_dir="logs"):
        l = Logger(); l.clear_handlers(); l.set_level(LogLevel.DEBUG)
        l.add_handler(ConsoleHandler(formatter=ColoredFormatter()))
        l.add_handler(RotatingFileHandler(
            os.path.join(log_dir, "app.log"),
            max_bytes=10_000_000, backup_count=5,
            formatter=PlainTextFormatter(),
            filter_chain=LevelFilter(LogLevel.WARNING)
        ))
        return l

    @staticmethod
    def create_test_logger():
        l = Logger(); l.clear_handlers()
        h = InMemoryHandler(); l.add_handler(h); return l, h

Interview Q&A

Full Interview Q&A

Click each question to reveal the model answer

Why use Singleton for Logger? What's the risk? ›

Why: A single Logger instance gives a centralized, consistent log stream across the entire app. Multiple logger instances would produce disconnected streams, making it impossible to enforce global level filters or see logs in chronological order.

Risk: Singletons make unit testing harder — the same instance persists between tests. Mitigate this by providing a clear_handlers() method and using InMemoryHandler in tests so you can assert what was logged without touching the real singleton.

Also mention: Global state introduces coupling. If you're building a library (not an application), expose a Logger factory instead of a hardcoded Singleton so library users can inject their own logger.

Explain double-checked locking. Why is it needed? ›

Without double-checked locking, a naive Singleton would either: (1) check if _instance is None without a lock — two threads can both see None and both create an instance (race condition), or (2) acquire a lock on every single call — expensive even after the instance exists.

Double-checked locking solves both: the outer if is a fast path that avoids the lock entirely once the instance exists. The inner if inside the lock catches the race where two threads both passed the outer check. This way the lock is only acquired when the instance might not yet exist.

In Python specifically, the GIL gives some protection, but you should still use the lock to be safe across implementations (PyPy, Jython, etc.).

What is the difference between the logger-level filter and per-handler filters? ›

The logger-level (set_level()) is a global gate. Any record below this level is dropped immediately, before a LogRecord is even created. This is for performance — no point creating objects for DEBUG messages in production.

Per-handler filters are finer-grained, applied after the record is created. They let each handler accept different subsets. Example: logger level is DEBUG (pass everything through), but the file handler only accepts WARNING+, and the remote handler only accepts ERROR+.

Think of it as: logger level = global pre-filter; handler filters = per-destination filters.

Why Chain of Responsibility for filters instead of a simple list? ›

A simple list works fine, but Chain of Responsibility gives you short-circuit evaluation (the chain stops as soon as one filter rejects the record) and composability (each filter only knows about its own check and who's next).

More importantly, it follows the Open/Closed Principle — adding a new filter doesn't change existing filter code at all. You just create a new Filter subclass and insert it into the chain.

In a simple list approach, you'd need to know and modify the list structure. In the chain, each filter is self-contained.

How would you make logging non-blocking? Why does it matter? ›

Use a QueueHandler: when logger.error() is called, it only puts a message string into an in-memory queue.Queue (microseconds). A background daemon thread drains the queue and does the actual I/O.

Why it matters: File I/O takes ~1–5ms. Network I/O takes 10–100ms. If your app logs 1000 events/second, blocking on each one would consume 5–100 seconds of I/O time per second — clearly a bottleneck. Non-blocking logging keeps the hot path at ~1µs.

Trade-off: If the app crashes, in-flight logs in the queue may be lost. Mitigate with queue.join() in the shutdown sequence (wait for the queue to drain before exiting).

How do you test a logging system? How do you assert logs were written? ›

Use an InMemoryHandler — a handler that stores formatted log strings in a list instead of writing to disk or console. In your test, inject this handler, call the code under test, then assert on handler.contains("expected message").

Pattern:

logger, handler = LoggerFactory.create_test_logger()
my_service.do_something() # code that logs
assert handler.contains("connection failed") # assertion

This approach keeps tests fast (no disk I/O), deterministic, and isolated — clearing the in-memory handler between tests prevents state leakage.

How does the Decorator pattern apply to RotatingFileHandler? ›

RotatingFileHandler extends Handler (same interface as FileHandler) and wraps the file-writing behavior with additional rotation logic. Before each write, it checks if the file size exceeds the threshold. If so, it renames old files and starts a new one — then delegates the actual write to the same open/write/close pattern as FileHandler.

The key Decorator property: it adds behavior transparently — callers just see a Handler and call handle(). The rotation is completely invisible to them.

This follows OCP: we didn't modify FileHandler at all. We just created a new class that adds new behavior.

What happens if a handler throws an exception? Should it crash the app? ›

Never. Logging is a support concern — it must never affect application correctness. If the remote log server is down, or the disk is full, the app should continue running normally. Only the logging for that handler fails.

Implementation: wrap each handler.handle(record) call in a try/except in Logger.log(). Catch all exceptions and print to stderr (the last resort that doesn't depend on the logging system). Never re-raise.

Similarly, in RemoteHandler._write(), wrap the HTTP call in try/except and fail silently (or print to stderr).

How would you extend this system to support named loggers (like Python's logging module)? ›

Instead of a single Singleton, maintain a registry (a dict) mapping logger names to Logger instances. Logger.get_logger("user.service") checks the registry; if not found, creates a new one with the appropriate parent based on the dot-separated hierarchy.

Hierarchical propagation: a log from "user.service" also propagates to "user" and then the root logger, unless propagate=False is set. This lets you configure handlers at the root that catch everything, and add specific handlers for particular modules.

This is exactly how Python's built-in logging module works: logging.getLogger("myapp.db") creates or retrieves a named logger.

What SOLID principles does this design follow? ›

S — Single Responsibility: Logger logs. Formatter formats. Handler writes. Filter filters. Each class has exactly one reason to change.

O — Open/Closed: Add a new handler, formatter, or filter by creating a new subclass — zero changes to existing code.

L — Liskov Substitution: Any Handler subclass can replace any other. Any Formatter subclass can replace any other. Logger only calls handle() and format().

I — Interface Segregation: Formatter has one method (format). Filter has one method (passes). Handler has one public method (handle). No fat interfaces.

D — Dependency Inversion: Logger depends on the abstract Handler, not concrete ConsoleHandler. Handler depends on abstract Formatter, not PlainTextFormatter.

Trade-offs

Design Trade-offs to Discuss

Mention these proactively — it shows senior-level thinking

Sync vs Async logging
Sync: simpler, guaranteed delivery, but blocks on I/O.
Async (QueueHandler): near-zero overhead, but may lose logs on crash. Production choice: async for high-throughput, sync for audit logs.
Singleton vs Dependency Injection
Singleton is convenient but couples code to global state. DI (pass logger as parameter) makes code easier to test. For libraries, always prefer DI. For applications, Singleton is acceptable.
Log levels as Enum vs int constants
IntEnum is ergonomic (level > LogLevel.INFO). Plain Enum is safer (no accidental arithmetic). In interviews, mention both and choose IntEnum for readability.
File open per-write vs persistent file handle
Opening per write: simple, safe across rotation and crashes. Persistent handle: faster (no open/close overhead). In production, use a buffered persistent handle with explicit flush().
Size-based vs time-based rotation
Size-based: predictable file sizes, good for disk management. Time-based: easy to correlate logs with incidents by date. Many production systems use both: rotate at midnight OR when file hits 100MB, whichever comes first.
Queue full: drop vs block
put_nowait() drops logs when queue is full (prefer this — never slow the app). put() blocks (bad for performance). Always drop with a stderr warning, and size the queue generously (10k–100k items).
Final tip for your interview: After presenting the design, proactively say "I chose X over Y because Z — the trade-off is..." This single habit separates strong candidates from average ones. Interviewers are evaluating your reasoning, not just your code.
